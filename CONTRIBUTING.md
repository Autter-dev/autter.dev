# Contributing

Thanks for wanting to get involved.

## Code contributions

Not yet. The codebase is moving fast and the architecture is still changing week to week. Opening it up to external code right now would create more confusion than value for both sides.

This will change. When it does, this file will have the full guide.

## What you can do right now

**Talk to us.** The [Discussions tab](https://github.com/autter-dev/autter/discussions) is the most useful place to contribute at this stage. Specifically:

- Found a gap in the approach? [Say so.](https://github.com/autter-dev/autter/discussions/categories/ideas)
- Have a question about how something is planned to work? [Ask it.](https://github.com/autter-dev/autter/discussions/categories/q-a)
- Something in the design looks wrong to you? [Open an issue.](https://github.com/autter-dev/autter/issues)

Maintainers and open source contributors who work with AI-generated code every day have the most useful perspective on this product. If that is you, your feedback shapes what gets built.

## Co-founder

If you are a technical or marketing co-founder and this problem space interests you, the [Discussions tab](https://github.com/autter-dev/autter/discussions/categories/general) is a reasonable place to start.
